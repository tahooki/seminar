Chatbot A2UI · Product Overview

질문을 업무 화면으로 바꾸는 채팅 UI

Chatbot A2UI는 사용자의 자연어 요청을 등록된 API 호출과 화면 템플릿 선택으로 연결해, 채팅 안에 조회 결과 화면을 표시합니다.

chatbot-a2ui.product

어디에 붙나요?

기존 Agent/API는 그대로 두고, 조회 결과 화면만 A2UI로 붙입니다.

고객 서비스 Agent가 기존 API를 호출하면, A2UI Admin MCP가 API 결과와 등록된 템플릿을 비교해 채팅 안 업무 화면으로 표시합니다.

01

고객 서비스 Agent가 기존 API를 호출합니다.

의도 판단, 권한 확인, 업무 데이터 조회는 기존 고객 서비스 흐름을 유지합니다.

02

A2UI Admin MCP가 템플릿을 선택합니다.

API 결과와 등록된 템플릿 계약을 비교해 표, 카드 목록, 프로필 중 맞는 화면을 고릅니다.

03

Renderer가 채팅 안에 표시합니다.

변환된 화면 데이터는 사용자가 보는 조회 결과 화면으로 렌더링됩니다.

어떤 화면으로 보여주나요?

같은 배포 데이터도 목적에 따라 다른 화면으로 보여줍니다.

목록 비교는 표, 빠른 훑어보기는 카드 목록, 한 건 분석은 프로필 화면으로 나뉩니다. 아래 화면은 등록된 템플릿으로 조회 결과를 표시하는 예시입니다.

표 화면

여러 행을 비교해야 하는 요청은 표 화면으로 표시됩니다.

카드 목록 화면

몇 개 항목을 빠르게 훑어보는 요청은 카드 목록으로 바뀝니다.

프로필 화면

한 건을 자세히 보는 요청은 프로필 중심 화면으로 표시됩니다.

어떻게 작동할까요?

고객 Agent가 API를 호출하고, A2UI가 결과 화면을 만듭니다.

1-3단계는 고객 서비스의 기존 Agent/API 흐름이고, 4-8단계부터 A2UI Admin MCP와 Renderer가 조회 결과를 화면화합니다.

누가 관리하고, 누가 사용하나요?

고객 서비스는 Agent와 API를 유지하고, A2UI는 결과 화면을 맡습니다.

제품의 핵심은 역할 분리입니다. 고객 서비스는 기존 Agent와 API를 유지하고, A2UI Library Team은 컴포넌트와 Admin MCP 템플릿, Renderer를 관리합니다.

어디까지 확장할 수 있나요?

템플릿 선택에서 시작해, 패턴 생성과 데이터 기반 UI로 확장됩니다.

현재 A2UI는 Admin에 미리 등록한 화면 템플릿을 Agent가 선택해 보여주는 구조입니다. 같은 기반 위에서 정해진 패턴 안의 UI 생성, 더 나아가 사용 데이터 기반 화면 제안까지 확장할 수 있습니다.
