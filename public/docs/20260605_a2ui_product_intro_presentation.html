Chatbot A2UI · Product Overview

질문을 업무 화면으로 바꾸는 채팅 UI

Chatbot A2UI는 사용자의 자연어 요청을 데이터 조회, 화면 선택, 액션 처리까지 연결해 채팅 안에서 바로 사용할 수 있는 업무 UI로 응답합니다.

무엇이 달라지나요?

질문 하나가 데이터 조회와 조작 가능한 화면으로 이어집니다.

A2UI는 질문을 이해하는 Agent, 화면 규칙을 관리하는 Admin, 실제 화면을 그리는 Renderer를 하나의 제품 경험으로 묶습니다.

01

질문이 곧 조회 조건이 됩니다.

문장 안의 기간, 환경, 상태 같은 단서를 읽어 바로 조회 조건으로 바꿉니다.

02

Agent가 데이터 모양을 보고 알맞은 화면을 고릅니다.

여러 행은 표로, 몇 개 항목은 카드 목록으로, 한 건 상세는 프로필 화면으로 구성합니다.

03

채팅 안에서 바로 조작합니다.

다음 페이지, 새로고침 같은 액션도 같은 화면 안에서 바로 처리합니다.

어떤 화면으로 보여주나요?

같은 배포 데이터도 목적에 따라 다른 화면으로 보여줍니다.

목록 비교는 표, 빠른 훑어보기는 카드 목록, 한 건 분석은 프로필 화면으로 나뉩니다. 아래 화면은 Agent의 선택 결과를 보여주는 예시입니다.

표 화면

여러 행을 비교해야 하는 요청은 페이지 이동이 가능한 표로 표시됩니다.

카드 목록 화면

몇 개 항목을 빠르게 훑어보는 요청은 카드 목록으로 바뀝니다.

프로필 화면

한 건을 자세히 보는 요청은 프로필 중심 화면으로 표시됩니다.

어떻게 작동할까요?

Agent가 Admin 템플릿을 MCP로 호출하고, Renderer가 화면에 표시합니다.

사용자 메시지가 들어오면 Chatbot Agent는 요청 의도를 판단하고, Admin MCP Tool을 통해 조회와 화면 구성을 진행합니다. 마지막에는 채팅 화면 렌더러가 검증된 컴포넌트로 결과를 표시합니다.

어떻게 연동하나요?

코드는 네 지점으로 연결됩니다.

제품 설명 이후에는 Admin MCP Tool 등록, Agent SDK 호출, Chatbot Agent의 화면 설계도 처리, 채팅 화면 렌더러 연결만 확인하면 됩니다.

// Admin MCP Tool
server.tool(
  "a2ui.runDataQuery",
  "자연어 데이터 조회 요청을 화면 설계도로 변환합니다.",
  async ({ input, history }) => {
    const result = await runDataA2UIQuery({
      input,
      history
    });

    return {
      content: [{
        type: "text",
        text: JSON.stringify(result)
      }]
    };
  }
);
// Agent SDK
export async function runDataQuery(client, options) {
  return await client.callTool(
    "a2ui.runDataQuery",
    {
      input: options.input,
      history: options.history ?? []
    }
  );
}
// Chatbot Agent entry
const result = await runDataQuery(client, {
  input,
  history
});

if (result.type === "surface") {
  return {
    surface: normalizeMcpSurface(result.surface)
  };
}
// Chatting UI Renderer
function ChatMessage({ message }) {
  if (message.surface) {
    return (
      <SurfaceRenderer
        surface={message.surface}
        onAction={handleSurfaceAction}
      />
    );
  }

  return <MarkdownMessage content={message.content} />;
}

누가 관리하고, 누가 사용하나요?

우리가 관리하고, 사용자는 사용만 하도록 합니다.

제품의 핵심은 역할 분리입니다. 라이브러리 개발팀은 UI 부품과 Admin 규칙을 관리하고, 최종 사용자는 자연어로 요청한 뒤 결과 화면을 사용합니다.

Library Team

관리하는 것

  • 재사용 가능한 Component UI
  • API와 데이터 연결 규칙
  • 화면 설계도 계약과 액션 정의
  • 채팅 화면 렌더러

End User

사용하는 것

  • 자연어 질문 입력
  • MCP나 라이브러리로 쉽게 연동된 업무 화면 사용
  • 응답 영역에 표시된 업무 화면 확인
  • 다음 페이지, 새로고침 같은 화면 안 액션 사용
  • UI 종류나 API 이름은 몰라도 되는 사용 경험

어디까지 확장할 수 있나요?

템플릿 선택에서 시작해, 패턴 생성과 데이터 기반 UI로 확장됩니다.

현재 A2UI는 Admin에 미리 등록한 화면 템플릿을 Agent가 선택해 보여주는 구조입니다. 같은 기반 위에서 정해진 패턴 안의 UI 생성, 더 나아가 사용 데이터 기반 화면 제안까지 확장할 수 있습니다.
