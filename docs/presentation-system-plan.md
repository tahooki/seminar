# Seminar Presentation System Plan

## 1. 프로젝트 목적

이 프로젝트는 세미나 발표를 위해 만든 단일 페이지 중심의 발표자료 제작 프로젝트다.
일반 문서나 블로그가 아니라, 발표자가 브라우저 한 화면에서 흐름을 넘기며 설명할 수 있는 HTML 기반 발표자료를 만드는 데 초점을 둔다.

핵심 목표는 발표 내용을 매번 처음부터 꾸미는 것이 아니라, 미리 구축해 둔 디자인 시스템과 발표용 컴포넌트를 조합해서 빠르게 완성도 높은 발표 페이지를 만드는 것이다.

## 2. 제품 방향

- 하나의 Next.js 페이지 안에서 발표자료를 구성한다.
- 발표 슬라이드는 HTML, CSS, React 컴포넌트로 그린다.
- 코드 블록, 다이어그램, 비교표, 단계 설명, 인용, 핵심 문장 같은 세미나용 요소를 재사용 가능한 컴포넌트로 제공한다.
- 한글과 영어가 섞인 발표에서도 글자가 편안하게 읽히도록 타이포그래피를 먼저 잡는다.
- 발표 화면은 노션 문서처럼 평평하지 않고, 약간 둥근 고딕 계열의 부드러운 인상을 가진다.
- 발표자는 개발자가 아니어도 최소한의 수정으로 내용을 바꿀 수 있어야 한다.

## 3. 디자인 키워드

- 편안한 고딕
- 약간 둥근 인상
- 넓은 행간과 안정적인 자간
- 발표장에서 멀리서도 읽히는 대비
- 과한 장식보다 명확한 정보 구조
- 한국어와 영어가 함께 있을 때 어색하지 않은 리듬

## 4. 타이포그래피 제안

### 기본 폰트 후보

1. `Pretendard`
   - 한글과 영어 모두 균형이 좋다.
   - 웹 서비스와 발표자료 모두에 잘 어울리는 현대적인 고딕이다.
   - 너무 딱딱하지 않으면서 정보 전달력이 좋다.

2. `SUIT`
   - 한글 발표자료에 잘 맞는 단정한 인상이다.
   - 자폭과 높이가 안정적이라 긴 설명문에도 적합하다.

3. `LINE Seed Sans KR`
   - 조금 더 둥글고 친근한 인상을 줄 수 있다.
   - 발표의 분위기를 부드럽게 만들고 싶을 때 잘 맞는다.

### 권장 방향

초기 기본값은 `Pretendard`로 둔다.
프로젝트가 원하는 "살짝 둥근 고딕"의 감성을 더 강하게 가져가고 싶다면 제목과 강조 문장에 `LINE Seed Sans KR` 계열을 실험한다.

권장 폰트 스택:

```css
font-family:
  "Pretendard",
  "LINE Seed Sans KR",
  "SUIT",
  "Apple SD Gothic Neo",
  "Noto Sans KR",
  system-ui,
  sans-serif;
```

### 발표용 글자 크기 기준

- 큰 제목: 56px-72px
- 섹션 제목: 40px-52px
- 본문: 24px-32px
- 보조 설명: 18px-22px
- 코드: 18px-24px

발표자료는 일반 웹페이지보다 보는 거리가 멀기 때문에 본문도 크게 잡는다.
모바일 대응보다 발표 화면의 가독성을 우선하되, 브라우저 폭이 줄어들 때는 슬라이드가 자연스럽게 세로 흐름으로 변하도록 한다.

## 5. 색상과 시각 톤

### 기본 톤

- 배경: 따뜻한 흰색 또는 아주 옅은 회색
- 본문: 검정에 가까운 짙은 회색
- 보조 텍스트: 중간 회색
- 강조색: 선명하지만 과하지 않은 블루, 그린, 코랄 중 1-2개
- 코드 영역: 밝은 배경에서는 옅은 회색, 어두운 코드 블록에서는 고대비 다크 톤

### 디자인 원칙

- 한 화면에 너무 많은 색을 쓰지 않는다.
- 강조색은 버튼보다 키워드, 라벨, 진행 단계, 다이어그램 노드에 우선 사용한다.
- 배경 장식은 최소화하고, 레이아웃과 타이포그래피로 발표의 밀도를 만든다.
- 카드 모서리는 8px 전후로 제한해서 둥글지만 유치하지 않은 인상을 유지한다.

## 6. 발표자료 디자인 시스템 구성

### Layout

- `Presentation`
  - 전체 발표 페이지를 감싸는 루트 레이아웃
  - 발표 폭, 배경, 키보드 네비게이션, 현재 슬라이드 상태를 관리

- `Slide`
  - 하나의 발표 화면 단위
  - 기본 비율은 `16:9`
  - 필요하면 긴 설명용 `ScrollSlide`를 별도로 둔다.

- `Section`
  - 여러 슬라이드를 묶는 챕터 구분
  - 발표 흐름상 큰 전환을 보여준다.

### Content Components

- `TitleSlide`
  - 발표 제목, 부제, 발표자, 날짜

- `AgendaSlide`
  - 발표 순서

- `StatementSlide`
  - 핵심 문장 하나를 크게 보여주는 슬라이드

- `TextSlide`
  - 제목과 설명문 중심의 기본 슬라이드

- `CompareSlide`
  - 두 개 이상의 개념을 비교하는 슬라이드

- `CodeSlide`
  - 코드 블록과 설명을 함께 보여주는 슬라이드

- `DiagramSlide`
  - Mermaid 또는 커스텀 SVG/HTML 다이어그램을 보여주는 슬라이드

- `QuoteSlide`
  - 인용문, 핵심 코멘트, 회고 문장

- `SummarySlide`
  - 발표 결론과 기억해야 할 항목

### Visual Components

- `Badge`
- `Callout`
- `Metric`
- `StepList`
- `Timeline`
- `ConceptMap`
- `ProcessFlow`
- `KeyboardKey`
- `InlineCode`

이 컴포넌트들은 발표자료 안에서 반복적으로 쓰이는 시각 언어를 통일한다.

## 7. Mermaid.js 활용 계획

발표자료에는 복잡한 시스템 구조, 흐름, 상태 전이, 시퀀스 설명이 자주 등장한다.
이를 위해 Mermaid.js를 설치하고 React 컴포넌트로 감싸서 사용한다.

### 목표

- Markdown처럼 간단한 문자열로 다이어그램을 작성한다.
- 다이어그램은 슬라이드 안에서 자동으로 렌더링된다.
- 발표 테마에 맞게 Mermaid 색상과 폰트를 커스터마이즈한다.
- 서버 렌더링 이슈를 피하기 위해 클라이언트 컴포넌트로 분리한다.

### 우선 지원할 다이어그램

- Flowchart
- Sequence Diagram
- State Diagram
- Gantt Chart
- Entity Relationship Diagram

### 예시 API

```tsx
<MermaidChart
  chart={`
    flowchart LR
      A["문제 정의"] --> B["핵심 개념"]
      B --> C["예제"]
      C --> D["정리"]
  `}
/>
```

## 8. Napkin 스타일 시각화 툴킷

Mermaid.js는 정형 다이어그램에 강하지만, 발표자료에는 Napkin처럼 생각을 빠르게 시각화해 주는 부드러운 그림 언어도 필요하다.
이 프로젝트는 "문장을 넣으면 발표용 시각 자료로 바꿀 수 있는" 경험을 목표로 삼고, 여러 시각화 방식을 발표용 컴포넌트로 감싸서 제공한다.

### 핵심 방향

- 발표자가 다이어그램 라이브러리의 세부 문법보다 "무엇을 보여줄지"에 집중하게 한다.
- 정형 다이어그램, 손그림 스타일, 마인드맵, 데이터 차트, 인터랙티브 노드 맵을 상황에 맞게 고를 수 있게 한다.
- 모든 시각화는 프로젝트의 폰트, 색상, 둥근 모서리, 여백 규칙을 따른다.
- 복잡한 원본 문법은 컴포넌트 내부에 숨기고, 발표자료 작성자는 간단한 데이터 구조로 사용한다.

### 우선 도입 후보

1. `mermaid`
   - Flowchart, Sequence Diagram, State Diagram, Gantt 같은 정형 다이어그램 담당
   - 기술 세미나의 기본 흐름과 시스템 설명에 적합

2. `roughjs`
   - 손으로 그린 듯한 선, 박스, 원, 화살표를 만드는 스케치 스타일 담당
   - Napkin 같은 친근한 시각화 감성을 만드는 핵심 후보

3. `markmap`
   - Markdown 목록을 마인드맵으로 변환
   - 개념 지도, 발표 목차, 문제 분해, 기술 키워드 관계를 보여줄 때 사용

4. `reactflow`
   - 노드와 엣지 기반의 인터랙티브 다이어그램 담당
   - 아키텍처 맵, 에이전트 워크플로우, 데이터 파이프라인, 의존성 그래프에 적합

5. `@observablehq/plot`
   - 수치와 비교 데이터를 발표용 차트로 렌더링
   - 성능 비교, 설문 결과, 로그 분석, 전후 변화 시각화에 사용

### 나중에 검토할 후보

- `d2`
  - Mermaid보다 정돈된 시스템 구조도를 만들고 싶을 때 검토

- `@excalidraw/excalidraw`
  - 발표 페이지 안에 직접 그리는 화이트보드 편집 경험이 필요할 때 검토

- `tldraw`
  - 무한 캔버스 기반의 인터랙티브 설명 화면이 필요할 때 검토

초기 버전에서는 편집기까지 내장하기보다, 발표자가 작성한 데이터와 텍스트를 정해진 시각 컴포넌트로 렌더링하는 데 집중한다.

### 발표용 시각화 컴포넌트

- `SketchNote`
  - 손그림 느낌의 메모 카드
  - 핵심 아이디어, 질문, 문제 정의를 부드럽게 강조

- `SketchFlow`
  - 손그림 화살표와 박스로 만든 흐름도
  - Mermaid보다 덜 딱딱한 설명용 프로세스에 사용

- `CycleDiagram`
  - 반복되는 루프나 운영 사이클을 원형 구조로 표현
  - 예: Paste -> Generate -> Select -> Customize -> Collaborate -> Export

- `ProcessFlow`
  - 순차적인 단계를 좌우 또는 상하 흐름으로 표현
  - 발표 중 "지금 어디를 설명하는지"를 보여주기 좋다.

- `ConceptMap`
  - 개념 사이의 관계를 노드와 선으로 표현
  - 기술 키워드, 도메인 개념, 문제 구성 요소를 연결

- `MindMap`
  - 하나의 주제에서 여러 하위 개념이 뻗는 구조
  - 발표 목차나 개념 정리에 사용

- `SystemMap`
  - 서비스, 모듈, 외부 API, 데이터 저장소의 연결을 표현
  - 기술 세미나의 아키텍처 설명에 사용

- `CompareMatrix`
  - 여러 선택지를 기준별로 비교하는 표형 시각화
  - 도구 비교, 전략 비교, before/after 비교에 사용

- `Timeline`
  - 시간 순서, 제품 발전, 장애 대응 흐름을 표현
  - 날짜가 중요한 발표에 사용

- `DataChart`
  - 막대, 선, 점, 영역 차트 등 데이터 중심 시각화
  - 숫자와 추세를 설명해야 하는 슬라이드에 사용

### 사용 예시

```tsx
<CycleDiagram
  title="Napkin 스타일 작성 흐름"
  center="Idea"
  items={[
    "Paste",
    "Generate",
    "Select",
    "Customize",
    "Collaborate",
    "Export",
  ]}
/>
```

```tsx
<SketchFlow
  items={[
    { title: "문장", description: "발표자가 적은 원본 아이디어" },
    { title: "구조화", description: "핵심 개념과 관계 추출" },
    { title: "시각화", description: "슬라이드에 맞는 그림으로 변환" },
  ]}
/>
```

### 구현 우선순위

1. `MermaidChart`
   - 정형 다이어그램의 기본 기반

2. `SketchNote`, `SketchFlow`
   - Rough.js 기반의 Napkin 감성 컴포넌트

3. `CycleDiagram`, `ProcessFlow`
   - 발표에서 가장 자주 쓰이는 흐름 시각화

4. `MindMap`, `ConceptMap`
   - 개념 설명과 목차 시각화

5. `SystemMap`, `DataChart`
   - 기술 발표와 수치 발표를 위한 확장 컴포넌트

## 9. 코드와 기술 스택 방향

현재 프로젝트는 Next.js App Router 기반이다.
발표자료 제작 경험을 단순하게 유지하기 위해 초반에는 페이지를 많이 늘리기보다, `app/page.tsx`에서 하나의 발표 흐름을 완성하는 방향으로 시작한다.

권장 구조:

```txt
app/
  layout.tsx
  page.tsx
  globals.css
src/
  presentation/
    components/
    data/
    theme/
```

### 설치 후보

- `mermaid`
  - 발표용 다이어그램 렌더링

- `roughjs`
  - 손그림 스타일의 박스, 선, 화살표 렌더링

- `markmap`
  - Markdown 기반 마인드맵 렌더링

- `reactflow`
  - 인터랙티브 노드 그래프와 시스템 맵 렌더링

- `@observablehq/plot`
  - 발표용 데이터 차트 렌더링

- `next-themes`
  - 라이트/다크 발표 테마가 필요할 때 사용

- `clsx`
  - 컴포넌트 variant 조합

- `tailwind-merge`
  - Tailwind 클래스 충돌 정리

- `lucide-react`
  - 발표자료 안에서 쓰기 좋은 선형 아이콘

### 코드 하이라이팅 후보

초기에는 CSS로 간단히 처리한다.
코드 설명이 많아지면 `shiki`를 도입해서 VS Code와 가까운 품질의 코드 하이라이팅을 제공한다.

## 10. 발표 작성 경험

발표자료는 개발자가 직접 React 컴포넌트를 조합하는 방식으로 시작한다.
이후 필요하면 JSON 또는 MDX 기반으로 확장할 수 있다.

초기 작성 예시:

```tsx
const slides = [
  {
    type: "title",
    title: "Next.js로 발표자료 만들기",
    subtitle: "HTML 기반 세미나 시스템 설계",
  },
  {
    type: "statement",
    statement: "발표자료는 문서가 아니라 흐름이다.",
  },
];
```

## 11. 구현 단계

### 1단계: 기본 발표 레이아웃

- 기본 폰트 적용
- 전체 배경과 색상 토큰 정의
- `Presentation`, `Slide` 컴포넌트 구현
- 샘플 슬라이드 5-7개 작성

### 2단계: 발표용 컴포넌트 세트

- 제목, 본문, 비교, 코드, 콜아웃, 단계 목록 컴포넌트 구현
- 발표자료 전용 spacing, radius, shadow, border 규칙 정리
