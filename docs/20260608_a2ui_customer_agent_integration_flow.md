# A2UI 고객 Agent 연동 메시지 정리

> 목적: Chatbot A2UI 제품 소개 슬라이드의 메시지를 "A2UI가 Agent 전체를 대체한다"가 아니라 "고객 서비스 Agent의 기존 API 호출 결과를 A2UI Admin MCP와 템플릿으로 화면화한다"로 고정한다.

## 검수 결론

이번 덱은 A2UI를 새로운 Agent 제품처럼 설명하면 안 된다. 고객 입장에서 이미 운영 중인 서비스 Agent와 API가 있고, A2UI는 그 결과를 채팅 안 업무 화면으로 바꿔주는 표시 계층으로 설명해야 한다.

따라서 슬라이드의 중심 문장은 다음처럼 잡는다.

> 기존 Agent/API는 그대로 두고, 조회 결과 화면만 A2UI로 붙입니다.

## 핵심 포지션

A2UI는 고객 서비스 Agent를 대체하지 않는다.

고객 서비스에는 이미 사용자 채팅, Agent 판단, 기존 API 호출 로직이 있다. A2UI는 그 뒤에 붙어서 API 조회 결과를 Admin MCP에 등록된 UI 템플릿과 연결하고, 채팅 안에 업무 화면으로 표시한다.

한 줄로 말하면 다음과 같다.

> 기존 Agent와 API 로직은 고객 서비스에 두고, A2UI는 조회 결과를 등록된 화면 템플릿으로 표시합니다.

## 1번 슬라이드 메시지

첫 슬라이드는 A2UI가 전체 Agent를 대신하는 그림이 아니라, 고객 서비스의 기존 흐름 뒤에 A2UI가 붙는 그림이어야 한다.

### 권장 흐름

```text
사용자 채팅
  -> 고객 서비스 Agent가 기존 API 호출
  -> A2UI Admin 템플릿 MCP 호출
  -> A2UI 챗봇 조회 결과 표시
```

### 화면 문구 후보

- 사용자 채팅
- 고객 서비스 Agent / 기존 API
- A2UI Admin 템플릿 MCP
- A2UI 챗봇 조회 결과

### 핵심 설명

사용자는 고객 서비스의 채팅에 질문한다. 고객 서비스 Agent는 기존 로직대로 의도를 판단하고 API를 호출한다. 이후 API 결과와 요청 맥락을 A2UI Admin MCP에 넘긴다. A2UI는 등록된 표, 카드 목록, 프로필 템플릿 중 맞는 템플릿을 선택하고, Renderer가 채팅 안에 조회 결과 화면을 표시한다.

### 이미지 구성 메모

- 왼쪽은 고객 서비스 영역, 오른쪽은 A2UI 영역으로 나눈다.
- 고객 서비스 영역에는 `사용자 채팅`, `고객 서비스 Agent`, `기존 API`를 둔다.
- A2UI 영역에는 `Admin 템플릿 MCP`, `Renderer`, `조회 결과 화면`을 둔다.
- 화살표는 "질문 -> API 결과 -> 템플릿 매핑 -> 화면 표시" 흐름으로만 보여준다.
- "A2UI가 API를 고른다"는 인상을 주는 문구는 넣지 않는다.

## A2UI 작동 순서

작동 순서 슬라이드는 "A2UI가 처음부터 API를 고른다"가 아니라, "기존 고객 Agent가 API 호출까지 수행하고, 그 결과를 A2UI가 화면 템플릿으로 변환한다"는 순서로 보여준다.

### 8단계

1. 사용자 질문
2. Agent 판단
3. API 호출
4. A2UI Admin MCP 호출
5. API 데이터와 UI 템플릿 데이터 비교
6. 템플릿 선택
7. 데이터 변환
8. 화면 표시

### 단계별 의미

| 단계 | 책임 | 의미 |
| --- | --- | --- |
| 1 사용자 질문 | 고객 서비스 | 사용자가 채팅에서 업무 요청을 입력한다. |
| 2 Agent 판단 | 고객 서비스 Agent | 기존 Agent가 의도, 권한, 필요한 API를 판단한다. |
| 3 API 호출 | 고객 서비스 Agent | 기존 API를 호출해 업무 데이터를 가져온다. |
| 4 A2UI Admin MCP 호출 | 고객 서비스 Agent -> A2UI | API 결과와 요청 맥락을 넘겨 화면 구성을 요청한다. |
| 5 API 데이터와 UI 템플릿 데이터 비교 | A2UI Admin MCP | API 응답 구조와 등록된 템플릿 계약을 비교한다. |
| 6 템플릿 선택 | A2UI Admin MCP | 표, 카드 목록, 프로필 등 등록된 템플릿 중 하나를 선택한다. |
| 7 데이터 변환 | A2UI Admin MCP | 선택된 템플릿에 맞게 columns, rows, fields, labels 등을 구성한다. |
| 8 화면 표시 | A2UI Renderer | 채팅 안에 조회 결과 화면을 표시한다. |

### 영역 구분

- 1-3번은 고객 서비스의 기존 Agent/API 영역이다.
- 4번은 고객 서비스가 A2UI를 붙이는 연동 지점이다.
- 5-8번은 A2UI가 맡는 화면화 영역이다.

슬라이드에서는 1-3번 카드 묶음에 `고객 서비스 기존 흐름`, 4-8번 카드 묶음에 `A2UI 화면화 흐름` 라벨을 붙이면 메시지가 가장 분명하다.

## 책임 분리

### 고객 서비스 / 고객 Agent

- 사용자 채팅 입력을 받는다.
- Agent 판단을 수행한다.
- 기존 API 호출 로직을 유지한다.
- 업무 데이터와 권한을 관리한다.
- A2UI Admin MCP를 호출해 API 결과의 화면 구성을 요청한다.

### A2UI / Library Team

- 표, 카드 목록, 프로필 같은 재사용 가능한 Component UI를 만든다.
- Admin MCP를 제공한다.
- 템플릿 계약과 데이터 매핑 규칙을 관리한다.
- API 응답 데이터를 템플릿 입력 데이터로 변환한다.
- Renderer로 채팅 안에 업무 화면을 표시한다.

### 고객에게 강조할 점

- 고객은 기존 Agent와 API를 버릴 필요가 없다.
- 고객 서비스는 A2UI Admin MCP를 붙여 조회 결과를 화면으로 표시할 수 있다.
- 라이브러리팀은 컴포넌트와 템플릿 계약을 관리한다.
- 이용 서비스는 A2UI를 통해 표, 카드 목록, 프로필 같은 화면을 쉽게 재사용한다.

## 오해를 줄이는 문장

다음 문장은 슬라이드나 발표 멘트 어딘가에 반드시 들어가야 한다.

> A2UI는 고객 Agent를 대체하지 않고, 고객 Agent가 가져온 조회 결과를 등록된 화면 템플릿으로 표시합니다.

또는 더 짧게는:

> 기존 Agent/API는 그대로, 결과 화면만 A2UI로 표시합니다.

## 제거할 슬라이드

### 2번 슬라이드 제거

`질문 하나가 조작 가능한 화면으로 이어집니다` 슬라이드는 현재 메시지와 어긋난다. 이 슬라이드는 A2UI가 사용자의 질문에서 조회 조건과 화면 선택까지 전부 직접 수행하는 느낌을 줄 수 있다.

### 코드 네 지점 슬라이드 제거

`코드는 네 지점으로 연결됩니다` 슬라이드는 개발자 구현 설명이 강하다. 고객 설득용 제품 흐름에서는 "고객 Agent/API에 A2UI Admin MCP와 Renderer를 붙인다"는 역할 분리 설명으로 충분하다.

## 이미지 생성 방향

### 1번 슬라이드

4단계 큰 흐름으로 만든다.

```text
사용자 채팅 -> 고객 서비스 Agent/API -> A2UI Admin 템플릿 MCP -> A2UI 챗봇 조회 결과
```

고객 서비스 Agent 영역과 A2UI 영역을 시각적으로 분리한다. A2UI가 API를 직접 고르는 느낌이 아니라, 고객 Agent가 API 호출 결과를 넘기는 느낌을 준다.

### 작동 순서 슬라이드

8단계 카드 배열을 유지하되, 1-3번은 고객 서비스 Agent 영역으로 묶고 4-8번은 A2UI 영역으로 묶는다.

```text
[고객 서비스 Agent]
1 사용자 질문
2 Agent 판단
3 API 호출

[A2UI]
4 A2UI Admin MCP 호출
5 API 데이터와 UI 템플릿 데이터 비교
6 템플릿 선택
7 데이터 변환
8 화면 표시
```

### 책임 분리 슬라이드

`Library Team / End User` 구도보다 `고객 서비스 / A2UI Library Team` 구도가 더 적합하다.

왼쪽:

- 고객 서비스 Agent
- 기존 API
- 업무 데이터

오른쪽:

- A2UI Admin MCP
- UI 템플릿
- Renderer

가운데:

- API 결과 데이터
- 템플릿 매핑 요청

### 책임 분리 슬라이드 문구 후보

제목:

> 고객 서비스는 Agent와 API를 유지하고, A2UI는 결과 화면을 맡습니다.

보조 문장:

> 라이브러리팀은 컴포넌트와 Admin MCP 템플릿을 관리하고, 고객 서비스는 기존 Agent가 가져온 API 결과를 A2UI로 표시합니다.

## 한 줄 결론

A2UI는 고객 서비스 Agent가 가져온 API 결과를, 라이브러리팀이 관리하는 템플릿과 Renderer를 통해 채팅 안 업무 화면으로 바꿔주는 표시 계층이다.
